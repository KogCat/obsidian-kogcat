# KogCat

[English](README.md)

AI 的回答很快。KogCat 让它值得信任。

每个聊天模型都会给你一个听起来很有把握的回答。但它无法告诉你——这个回答是否建立在一个错误的假设上，是否绕过了真正的问题，或者正在把你带进一个它自己都看不见的坑。KogCat 对每条回答做一次校准，对照结构化知识库（认知偏误、逻辑谬误、决策框架）评估模型说了什么、漏掉了什么，然后在不替换原始回答、不打断你节奏的前提下，把值得注意的地方告诉你。

KogCat 没有自己的服务器。校准引擎以本地进程运行在你的机器上。插件不会读取你的 vault——只有你在对话面板里输入的内容才会被处理。你用哪个 LLM 服务商完全由你决定，KogCat 读取模型回答完成后的结果，在本地做校准。整个过程不需要账号、不需要订阅、也不需要把你的笔记交给任何第三方。

## 看看有什么不同

### 例子一 — 学习

你的问题：*"我每天读书 30 分钟，但感觉什么都记不住。我是不是应该在读的时候做更详细的笔记？"*

**没有 KogCat — 模型的回答：**

> 做详细笔记确实能提升记忆效果。你可以试试康奈尔笔记法：把页面分成线索区、笔记区和摘要区，边读边标注关键段落，再用自己的话重新写一遍。Anki 这样的间隔重复工具也能帮你强化已经记录的内容。

**有 KogCat — Quick Answer 顾问卡片：**

> **KogCat 建议先看这一点** ›
>
> 读书时做更详细的笔记，很可能让问题更严重，而不是更好。瓶颈不在于记录，而在于提取练习。学习研究持续表明：合上书、凭记忆把内容写出来（哪怕不完整），比任何在阅读过程中做笔记的方法都能产生更强的记忆效果。做更多笔记会增加"学到了"的感觉，但不会改变真正留下来的东西。
>
> 在改变笔记格式之前，先试一次：读完一章，关掉所有东西，凭记忆写一段粗略总结。看看你实际记住了多少，跟你以为自己记住了多少做个对比。

### 例子二 — 推进项目

你的问题：*"我有个东西两周后要交，但一直拖着没动。试过时间块规划，还是会在那段时间里去做别的事。怎么办？"*

**没有 KogCat — 模型的回答：**

> 拖延通常源于任务模糊或精力不足。试试把这个交付物拆成更小的子任务，分别安排时间。番茄工作法也很有帮助——先承诺专注 25 分钟，再休息一下。在专注时段里把手机放到另一个房间、开启网站屏蔽，效果会更好。

**有 KogCat — Quick Answer 顾问卡片：**

> **KogCat 建议先看这一点** ›
>
> 如果时间块规划已经不管用，在上面加番茄钟和屏蔽工具，是在用更多结构应对一个不同性质的问题。对一个有明确截止日期的具体任务持续回避，通常意味着两种情况之一：下一步行动不够具体（你知道目标，但不知道字面意义上的第一个动作是什么），或者这个交付物本身有某个让你一直在回避做决定的地方。
>
> 基于结构的方案解决的是注意力问题，不解决模糊或回避问题——结果往往是你在"工作"，但一直在处理项目里压力更低的部分。更快的问题是：这个交付物的下一个具体动作是什么？有没有某件事你一直在回避决定？

## 两种模式，一个插件

**Quick Answer（快速答案）** 保持你原有的对话节奏。模型正常回答，KogCat 在后台评估，只有当它的判断与模型存在明显差距时，才在回答下方显示顾问卡片。原始回答不会被悄悄替换，是否采纳由你决定。

**Advisor Answer（顾问答案）** 让 KogCat 成为主要回答者。模型依然在后台运行，但你看到的是 KogCat 整理后的结构化回答：明确结论、影响建议的前提条件、关键风险或边界、以及具体的下一步行动。适合用在"说错了代价很高"的问题上。

两种模式通过对话面板顶部的一个控件切换。

---

## 什么会被处理，什么不会

- **你的 vault：** KogCat 不会读取。插件无法访问你的笔记，除非你自己把内容粘贴到对话框里。
- **你的对话消息：** 发送给你选择的 LLM 服务商，和其他任何聊天插件一样。KogCat 看到的是模型回答完成后的结果，不介入消息传输过程。
- **校准过程：** 完全在本地引擎进程内运行。判断结果不离开你的机器。
- **LLM：** 你的 API Key，你的服务商，你来选。KogCat 在回答上面加一层，不替换、不代理模型。
- **引擎二进制文件：** 从固定的 GitHub Release 下载一次，运行前验证 sha256 完整性。不会静默更新。

## 环境要求

- Obsidian 1.0.0 或更高版本（仅桌面端，不支持移动端）。
- macOS（arm64 / x64）或 Windows x64。
- 至少一个支持服务商的 API Key（Anthropic、OpenAI、Google、Groq 等），在插件设置中填写。

## 安装

1. 在 Obsidian 社区插件市场搜索并安装 KogCat，或将 release 包解压到 `<vault>/.obsidian/plugins/kog-cat/`。
2. 启用插件。
3. 打开**设置 → KogCat**，填入服务商 API Key。
4. 打开对话面板。首次使用时，校准引擎会自动下载，安装前验证 sha256 完整性。后续启动复用缓存，无需重复下载。

## 使用方法

1. 从侧边栏图标或命令面板（`KogCat: Open chat`）打开对话面板。
2. 在面板顶部选择答案模式：**Quick Answer** 或 **Advisor Answer**。
3. 正常提问。
   - **Quick Answer：** 当 KogCat 有值得补充的内容时，回答下方出现顾问卡片；点击 **View KogCat Answer** 展开查看。
   - **Advisor Answer：** KogCat 的结构化回答直接作为助手回复显示。

没有单独的校准步骤，不需要手动触发，也不需要提前调参。
